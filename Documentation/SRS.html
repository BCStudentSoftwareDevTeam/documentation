Labor Status Forms - System Requirements Specification (SRS)

  1. New Labor Status Form interface must allow Admin:
    •  To create a primary labor status form for a student
    • To create a secondary labor status form for a student who already has an approved primary job
    • To create a labor status form for a break term
    • To create a primary overload form  
    • To create a multiple labor status forms for the different students with the same Labor Supervisor and Department in the same term.
    • To create a secondary Overload Form  
    • To create a form with WLS more than 5
    • To remove students from the table when creating one or multiple forms at the same time
    • To create notes for a student before submitting their Labor Status Form
    • Save the information for students that they have added and then submit LSF for them later
