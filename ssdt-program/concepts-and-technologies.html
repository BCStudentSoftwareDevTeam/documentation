Model View Controller

All of our applications use the Model View Controller (MVC) framework for developing applications. MVC is a common way to separate the application into three logical units (1) the data-handling and business logic (Model), (2) User Interface logic (View), (3) and input and request logic (Controller): 

Image Source: https://i.ytimg.com/vi/CPBqBA2qK8U/maxresdefault.jpg

Following the diagram above, we can see that the User interacts with the View. The User might take an Action in the View, which triggers the Controller. The Controller might need to request that the Model  Update data, so the Model then applies the business logic and makes the change in the database. After the change is made, the Controller is Notified of the change, which then Updates the View to the User.
